概要：

CircleCIと統合されているGitHubの組織は、以下のいずれかの統合タイプを持っており、これによって組織のメンバーの役割と権限がどのように管理されるかが決まります：

  • GitHub OAuthアプリ統合
  • GitHubアプリ統合

この形式のURLは、GitHub OAuthによって認証されたことを示します：https://app.circleci.com/pipelines/github/<組織名>

この形式のURLは、GitHubアプリによって認証されたことを示します：https://app.circleci.com/pipelines/circleci/UTxCZDiJ9MLGLC8hR1ZDmg

 

ユーザーの管理方法

  • GitHub OAuthユーザー

    • OAuthユーザーの役割と権限は、GitHubの組織ページから継承されます
    • GitHubで行われたユーザーの権限の変更がある場合はCircleCIにも反映されます
      • ユーザーの変更は、GitHubにおける組織のページ内、Peopleタブから行うことが可能です

  • GitHubアプリユーザー

    • 組織管理者として、CircleCIダッシュボードのサイドバーにあるOrganization Settingsへと移動します
      • Peopleをクリックします

追加の注意事項：

  • 現在、既存のCircleCIユーザーに対して、GH OAuth統合からGHアプリ統合への変換または変更をサポートしていません。
