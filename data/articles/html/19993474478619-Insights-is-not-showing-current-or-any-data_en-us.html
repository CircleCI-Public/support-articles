Problem:

When you trigger a build, the metrics of these recent workflows might not appear on the Insights page or be available via the API. This might result in the Insights page or API not returning any data or returning outdated data.

Insights is not a real-time reporting tool and therefore data might take up to 24 hours to appear.

Solutions:

  • Please verify if the workflow with the missing metrics has been triggered during the last 24 hours. The metrics are refreshed daily and they will appear within 24 hours. If your project did not have any workflows outside of the 24-hour window, the Insights page will appear empty during this time.
  • If you are using the API, please check if you set a query parameter of reporting-window=last-24-hours. If this is the case, please try increasing the reporting window.

Outcome:

The metrics should appear within 24 hours. If this is not the case, please feel free to reach out to Support for further investigation.
